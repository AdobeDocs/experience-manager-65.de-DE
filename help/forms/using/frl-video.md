# Videoskript: Einrichten einer funktionsbeschränkten Lizenzierung (FRL) für Adobe Acrobat auf AEM Forms

| Visuell | Titel | Sprecherskript |
|--------|-------|---------------|
| Titelbildschirm mit AEM Forms-Logo und FRL-Grafik | Einführung | Willkommen in diesem Tutorial zum Einrichten einer funktionsbeschränkten Lizenzierung (FRL) für Adobe Acrobat auf Ihrem AEM Forms-Server. Dieser Prozess ist von entscheidender Bedeutung, da Adobe von unbefristeten Lizenzen auf das neue FRL-Modell umstellt. |
| Zeitleiste mit EOL-Datumsangaben mit Pfeilen, die auf die URL-Transition zeigen | Hintergrund | Adobe Acrobat Classic-Versionen mit unbefristeten Lizenzen erreichen ihr Ende und erfordern einen Wechsel zu neueren Versionen. AEM Forms PDF Generator nutzt Adobe Acrobat für die Dokumentkonvertierung. Das Modell der eingeschränkten Lizenzierung mit eingeschränkten Funktionen bietet dieselben Funktionen, jedoch mit einem modernen Lizenzierungsansatz, der über die Adobe Admin Console verwaltet wird. |
| Checkliste der erforderlichen Elemente mit Kontrollkästchen und Symbolen | Voraussetzungen | Bevor wir beginnen, stellen wir sicher, dass Sie alles haben, was Sie benötigen: Zunächst einmal Zugriff auf Adobe Admin Console mit Systemadministratorrechten. Zweitens ein Benutzerkonto mit der Rolle „Bereitstellungsadministrator“ in der Admin Console. Drittens, lokale Administratorrechte auf dem Server, auf dem AEM Forms ausgeführt wird. Viertens, ein Windows 64-Bit-Betriebssystem auf Ihrem AEM Forms-Server. Fünftens, eine stabile Internetverbindung für die Lizenzaktivierung. Sie sollten auch mit der Benutzeroberfläche von Adobe Admin Console vertraut sein und Ihre Bereitstellungsarchitektur für AEM Forms verstehen. |
| Warnsymbol mit Sicherungsgrafik | Wichtiger Hinweis | Denken Sie daran, alle benutzerdefinierten Acrobat-Einstellungen zu sichern, bevor Sie beginnen, da Sie während dieses Vorgangs bestehende Acrobat-Installationen deinstallieren müssen. |
| Flussdiagramm mit nummerierten Schritten von 1-7 | Prozessübersicht | Dieser Prozess umfasst mehrere wichtige Schritte: Vorbereiten eines FRL-Pakets in der Admin Console, Gewähren von Download-Berechtigungen, Deinstallieren früherer Acrobat-Versionen, Installieren des neuen Acrobat Pro-Pakets, Bereitstellen des FRL-Pakets und Überprüfen der Installation. Lassen Sie uns die einzelnen Schritte durchgehen. |
| Screenshot der Adobe Admin Console-Benutzeroberfläche mit hervorgehobener Registerkarte „Pakete“ | Schritt 1: Admin Console-Vorbereitung | Melden Sie sich zunächst mit Systemadministratorrechten bei Adobe Admin Console an. Navigieren Sie zur Registerkarte Pakete und wählen Sie „Funktion der eingeschränkten Lizenz“ aus. Klicken Sie auf die Schaltfläche „Erste Schritte“, um mit der Erstellung Ihres Pakets zu beginnen. |
| Bildschirm „Paketkonfiguration erstellen“ mit hervorgehobenen Einstellungen | Package-Konfiguration | Konfigurieren Sie Ihr Paket mit den folgenden empfohlenen Einstellungen: Wählen Sie „Offline“ für die Aktivierungsmethode, &quot;PDF-Generierung“ für die Berechtigung und „Windows 64-Bit“ für die Plattform aus. Für die Programmauswahl behalten Sie nur die Lizenzdatei in den ausgewählten Programmen bei. Geben Sie Ihrem Paket einen beschreibenden Namen wie &quot;Acrobat FRL AEM Forms&quot; und erstellen Sie das Paket. |
| Registerkarte &quot;Admin Console-Benutzer“ mit Dialogfeld „Berechtigungen“ | Schritt 2: Download-Berechtigungen erteilen | Als Nächstes müssen Sie einen Benutzer erstellen oder identifizieren, der das Paket herunterladen wird. Navigieren Sie in der Admin Console zur Registerkarte „Benutzer“, wählen Sie Ihren Benutzer aus und weisen Sie ihm die Rolle „Bereitstellungs-Administrator“ zu. Dadurch können sie das von Ihnen erstellte FRL-Paket herunterladen. |
| Windows-Systemsteuerung mit Apps und Funktionen, die Acrobat anzeigen | Schritt 3: Vorherige Acrobat deinstallieren | Vor der Installation der neuen Version müssen Sie alle bestehenden Acrobat-Installationen vollständig entfernen. Öffnen Sie die Windows-Systemsteuerung, navigieren Sie zu Apps, suchen Sie nach Adobe Acrobat und klicken Sie auf Deinstallieren. Für eine gründliche Entfernung sollten Sie das Adobe Acrobat Cleaner Tool verwenden, insbesondere wenn Sie später im Prozess auf Probleme stoßen. |
| Adobe Acrobat DC-Downloadseite mit hervorgehobenem Installationsprogrammabschnitt | Schritt 4: Herunterladen und Installieren von Adobe Acrobat Pro | Laden Sie nach der Deinstallation das entsprechende Acrobat Pro-Installationsprogramm von der Seite Adobe Acrobat DC Downloads herunter. Aus Gründen der PDF Generator-Kompatibilität wird das Windows 32-Bit-Installationsprogramm empfohlen. |
| Installationsprozess mit Extraktions- und Einrichtungsschritten | Installationsprozess | Extrahieren Sie die heruntergeladene ZIP-Datei in einen Ordner, suchen Sie „Setup.exe“ und führen Sie sie als Administrator aus. Folgen Sie den Anweisungen auf dem Bildschirm, um die Installation abzuschließen. Öffnen Sie Acrobat Pro nach der Installation einmal, um alle Begrüßungsdialoge zu schließen und die Ersteinrichtung abzuschließen. |
| Registerkarte &quot;Admin Console-Pakete“ mit hervorgehobener Schaltfläche „Herunterladen“ | Schritt 5: Herunterladen des FRL-Pakets | Melden Sie sich jetzt mit dem Konto, für das Sie zuvor Download-Berechtigungen erteilt haben, bei der Adobe Admin Console an. Navigieren Sie zur Registerkarte Pakete , suchen Sie das FRL-Paket und laden Sie es auf Ihren AEM Forms-Server herunter. |
| Eingabeaufforderung als Administrator mit eingegebenen Befehlen | Schritt 6: Bereitstellen des Pakets | Extrahieren Sie das heruntergeladene Paket in ein Verzeichnis auf Ihrem Server. Öffnen Sie die Eingabeaufforderung als Administrator und navigieren Sie zum Extraktionsverzeichnis. |
| Befehlszeile, die den Aktivierungsbefehl mit hervorgehobener Syntax anzeigt | Aktivierungsbefehl | Führen Sie den Aktivierungsbefehl mit folgender Syntax aus: `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`. Ersetzen Sie den JSON-Dateinamen durch den genauen Dateinamen aus Ihrem Paket. Die Befehlsparameter sind: -p für die Plattform, -i für die Installation der Lizenz und -f für die Angabe des JSON-Dateipfads. Bei erfolgreicher Ausführung wird die Meldung „Vorgang erfolgreich abgeschlossen“ angezeigt. |
| Tabelle mit Befehlsparametern und Beschreibungen | Befehlsparameter | Der Parameter -p gibt die Plattform an und erkennt das Betriebssystem automatisch. Der Parameter -i weist das Tool an, die Lizenz zu installieren und zu aktivieren. Und der Parameter -f gibt den Pfad zur JSON-Lizenzdatei an. |
| AEM Forms-Admin-Benutzeroberfläche mit PDF Generator-Service | Schritt 7: Testen und Überprüfen | Testen Sie abschließend Ihre Installation, indem Sie die Admin-Oberfläche von AEM Forms öffnen und ein einfaches Dokument mithilfe des PDF Generator-Services in PDF konvertieren. Überprüfen Sie außerdem die Acrobat-Installation, indem Sie Acrobat Pro öffnen, zu Hilfe → Info navigieren und die Versionsnummer und den Lizenzstatus bestätigen. |
| Geteilter Bildschirm: Acrobat Info zum Dialogfeld und erfolgreiche PDF-Konversion | Überprüfung | Überprüfen Sie, ob die Lizenz in Acrobat als aktiviert angezeigt wird und ob PDF Generator Dokumente erfolgreich konvertieren kann. |
| Liste häufiger Fehler und Lösungen mit Symbolen | Tipps zur Fehlerbehebung | Wenn Probleme auftreten, überprüfen Sie diese häufigen Probleme: Stellen Sie sicher, dass Sie als Administrator eine Eingabeaufforderung ausführen, überprüfen Sie, ob der JSON-Dateiname richtig ist, überprüfen Sie, ob widersprüchliche Installationen vorliegen, und bestätigen Sie, dass Ihr Server über eine ordnungsgemäße Internetverbindung für die Aktivierung verfügt. |
| Zusätzliche Elemente zur Fehlerbehebung mit Auswahlfeldern | Weitere Fehlerbehebung | Überprüfen Sie außerdem auf Firewall-Einschränkungen, beschädigte Pakete oder in Konflikt stehende Adobe-IDs auf dem Server. Wenn alles andere fehlschlägt, wenden Sie sich mit detaillierten Fehlermeldungen an den Adobe-Support. |
| Erfolgsbildschirm mit Häkchen und Kontaktinformationen | Zusammenfassung | Sie haben jetzt erfolgreich die eingeschränkte Funktionslizenzierung für Adobe Acrobat auf Ihrem AEM Forms-Server eingerichtet. Wenn Sie zusätzliche Hilfe benötigen, wenden Sie sich an den Adobe-Support oder lesen Sie die detaillierte Dokumentation auf Experience League. |
