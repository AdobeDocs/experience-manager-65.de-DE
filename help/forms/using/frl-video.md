# Videoskript: Einrichten Lizenzierung mit Funktionseinschränkungen (FRL) für Adobe Acrobat auf AEM Forms

| Visuell | Titel | Sprecher-Skript |
|--------|-------|---------------|
| Titelbildschirm mit AEM Forms-Logo und FRL-Grafik | Einführung | Willkommen zu diesem Tutorial zum Einrichten einer Lizenzierung mit Funktionseinschränkungen (Feature Restricted Licensing, FRL) für Adobe Acrobat auf Ihrem AEM Forms-Server. Dieser Prozess ist von entscheidender Bedeutung, da Adobe von unbefristeten Lizenzen auf das neue FRL-Modell umstellt. |
| Timeline mit EOL-Datumsangaben und Pfeilen, die auf die URL-Transition zeigen | Hintergrund | Adobe Acrobat Classic-Versionen mit unbefristeten Lizenzen erreichen das Ende ihrer Lebensdauer und müssen auf neuere Versionen umgestellt werden. AEM Forms PDF Generator nutzt Adobe Acrobat für die Dokumentkonvertierung. Das Modell der Lizenzierung mit Funktionseinschränkungen bietet dieselben Funktionen, jedoch mit einem modernen Lizenzierungsansatz, der über die Adobe Admin Console verwaltet wird. |
| Checkliste der erforderlichen Elemente mit Kontrollkästchen und Symbolen | Voraussetzungen | Bevor wir beginnen, stellen wir sicher, dass Sie alles haben, was Sie benötigen. 1. Zugriff auf die Adobe Admin Console mit Systemadministratorrechten. 2. Ein Benutzerkonto mit der Rolle „Bereitstellungsadministrator“ in der Admin Console. 3. Lokale Administratorrechte auf dem Computer, auf dem AEM Forms ausgeführt wird 4. Ein Windows 64-Bit-Betriebssystem auf Ihrem AEM Forms-Server. 5. Eine stabile Internet-Verbindung für die Lizenzaktivierung Sie sollten auch mit der Benutzeroberfläche von Adobe Admin Console vertraut sein und Ihre Bereitstellungsarchitektur für AEM Forms verstehen. |
| Warnsymbol mit Sicherungsgrafik | Wichtiger Hinweis | Denken Sie daran, alle benutzerdefinierten Acrobat-Einstellungen zu sichern, bevor Sie beginnen, da Sie im Rahmen dieses Vorgangs bestehende Acrobat-Installationen deinstallieren müssen. |
| Flussdiagramm mit nummerierten Schritten von 1 bis 7 | Prozessüberblick | Dieser Prozess umfasst mehrere wichtige Schritte: Vorbereiten eines FRL-Pakets in der Admin Console, Gewähren von Download-Berechtigungen, Deinstallieren früherer Acrobat-Versionen, Installieren des neuen Acrobat Pro-Pakets, Bereitstellen des FRL-Pakets und Überprüfen der Installation. Lassen Sie uns die einzelnen Schritte durchgehen. |
| Screenshot der Adobe Admin Console-Benutzeroberfläche mit hervorgehobener Registerkarte „Pakete“ | Schritt 1: Vorbereitung der Admin Console | Melden Sie sich zunächst mit Systemadministratorrechten bei der Adobe Admin Console an. Navigieren Sie zur Registerkarte „Pakete“ und wählen Sie „Feature Restricted License“ aus. Klicken Sie auf die Schaltfläche „Jetzt loslegen“, um mit der Erstellung Ihres Pakets zu beginnen. |
| Bildschirm „Paketkonfiguration erstellen“ mit hervorgehobenen Einstellungen | Paketkonfiguration | Konfigurieren Sie Ihr Paket mit den folgenden empfohlenen Einstellungen: Wählen Sie „Offline“ für die Aktivierungsmethode, „PDF-Generierung“ für die Berechtigung und „Windows 64-Bit“ für die Plattform aus. Für die Programmauswahl behalten Sie nur die Lizenzdatei in den ausgewählten Anwendungen bei. Geben Sie Ihrem Paket einen beschreibenden Namen wie „Acrobat FRL AEM Forms“ und erstellen Sie das Paket. |
| Registerkarte „Admin Console-Benutzer“ mit Dialogfeld „Berechtigungen“ | Schritt 2: Download-Berechtigungen erteilen | Als Nächstes müssen Sie eine Person erstellen oder identifizieren, die das Paket herunterladen wird. Navigieren Sie in der Admin Console zur Registerkarte „Benutzer“, wählen Sie Ihre Benutzerin bzw. Ihren Benutzer aus und weisen Sie ihr bzw. ihm die Rolle „Bereitstellungs-Administrator“ zu. Dadurch können sie das von Ihnen erstellte FRL-Paket herunterladen. |
| Windows-Systemsteuerung mit Apps und Funktionen, die Acrobat anzeigen | Schritt 3. Vorherige Acrobat-Version deinstallieren | Vor der Installation der neuen Version müssen Sie alle bestehenden Acrobat-Installationen vollständig entfernen. Öffnen Sie die Windows-Systemsteuerung, navigieren Sie zu „Apps“, suchen Sie nach „Adobe Acrobat“ und klicken Sie auf „Deinstallieren“. Für eine gründliche Entfernung sollten Sie das Adobe Acrobat Cleaner Tool verwenden, insbesondere wenn Sie später im Prozess auf Probleme stoßen. |
| Seite der Adobe Acrobat DC-Downloads mit hervorgehobenem Installationsprogrammabschnitt | Schritt 4: Adobe Acrobat Pro herunterladen und installieren | Laden Sie nach der Deinstallation das entsprechende Acrobat Pro-Installationsprogramm von der Seite „Adobe Acrobat DC Downloads“ herunter. Aus Gründen der Kompatibilität mit PDF Generator wird das Windows 32-Bit-Installationsprogramm empfohlen. |
| Installationsprozess mit Extraktions- und Einrichtungsschritten | Installationsprozess | Extrahieren Sie die heruntergeladene ZIP-Datei in einen Ordner, suchen Sie „Setup.exe“ und führen Sie sie als Admin aus. Folgen Sie den Anweisungen auf dem Bildschirm, um die Installation abzuschließen. Öffnen Sie Acrobat Pro nach der Installation einmal, um alle Begrüßungsdialoge zu schließen und die Ersteinrichtung abzuschließen. |
| Registerkarte „Admin Console-Pakete“ mit hervorgehobener Schaltfläche „Herunterladen“ | Schritt 5: FRL-Paket herunterladen | Melden Sie sich jetzt mit dem Konto, für das Sie zuvor Download-Berechtigungen erteilt haben, bei der Adobe Admin Console an. Navigieren Sie zur Registerkarte „Pakete“, suchen Sie Ihr FRL-Paket und laden Sie es auf Ihren AEM Forms-Server herunter. |
| Eingabeaufforderung als Admin mit eingegebenen Befehlen | Schritt 6: Paket bereitstellen | Extrahieren Sie das heruntergeladene Paket in ein Verzeichnis auf Ihrem Server. Öffnen Sie die Eingabeaufforderung als Admin und navigieren Sie zum Extraktionsverzeichnis. |
| Befehlszeile, die den Aktivierungsbefehl mit hervorgehobener Syntax anzeigt | Aktivierungsbefehl | Führen Sie den Aktivierungsbefehl mit folgender Syntax aus: `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`. Ersetzen Sie den JSON-Dateinamen durch den genauen Dateinamen aus Ihrem Paket. Die Befehlsparameter sind: -p für die Plattform, -i für die Installation der Lizenz und -f für die Angabe des JSON-Dateipfads. Bei erfolgreicher Ausführung wird die Meldung „Vorgang erfolgreich abgeschlossen“ angezeigt. |
| Tabelle mit Befehlsparametern und Beschreibungen | Befehlsparameter | Der Parameter -p gibt die Plattform an und erkennt das Betriebssystem automatisch. Der Parameter -i veranlasst das Tool zur Installation und Aktivierung der Lizenz. und der Parameter -f gibt den Pfad zur JSON-Lizenzdatei an. |
| Admin-Benutzeroberfläche von AEM Forms mit dem PDF Generator-Service | Schritt 7: Testen und Überprüfen | Testen Sie abschließend Ihre Installation, indem Sie die Admin-Oberfläche von AEM Forms öffnen und ein einfaches Dokument mithilfe des PDF Generator-Services in PDF konvertieren. Überprüfen Sie außerdem die Acrobat-Installation, indem Sie Acrobat Pro öffnen, zu „Hilfe“ → „Info“ navigieren und die Versionsnummer und den Lizenzstatus bestätigen. |
| Geteilter Bildschirm: Info-Dialogfeld von Acrobat und erfolgreiche PDF-Konversion | Überprüfung | Überprüfen Sie, ob die Lizenz in Acrobat als aktiviert angezeigt wird und ob PDF Generator Dokumente erfolgreich konvertieren kann. |
| Liste häufiger Fehler und Lösungen mit Symbolen | Tipps zur Fehlerbehebung | Wenn Probleme auftreten, überprüfen Sie diese gängigen Probleme: Stellen Sie sicher, dass Sie eine Eingabeaufforderung als Admin ausführen, überprüfen Sie, ob der JSON-Dateiname richtig ist, überprüfen Sie, ob widersprüchliche Installationen vorliegen, und bestätigen Sie, dass Ihr Server über eine ordnungsgemäße Internet-Verbindung für die Aktivierung verfügt. |
| Zusätzliche Elemente zur Fehlerbehebung mit Auswahlfeldern | Weitere Fehlerbehebung | Überprüfen Sie außerdem auf Firewall-Einschränkungen, beschädigte Pakete oder in Konflikt stehende Adobe-IDs auf dem Server. Wenn dies alles nicht hilft, wenden Sie sich mit detaillierten Fehlermeldungen an den Adobe-Support. |
| Erfolgsbildschirm mit Häkchen und Kontaktinformationen | Zusammenfassung | Sie haben jetzt erfolgreich die eingeschränkte Funktionslizenzierung für Adobe Acrobat auf Ihrem AEM Forms-Server eingerichtet. Wenn Sie zusätzliche Hilfe benötigen, wenden Sie sich an den Adobe-Support oder lesen Sie die detaillierte Dokumentation auf Experience League. |
| Abschlussbildschirm mit Adobe-Logo und Credits | Vielen Dank | Vielen Dank, dass Sie sich dieses Tutorial angesehen haben. Weitere Informationen finden Sie in der Dokumentation zu Experience League. |
